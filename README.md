# 자기소개 A
Git과 GitHub의 브랜치, Pull Request, 셀프 리뷰, Merge 흐름을 연습하기 위한 자기소개 과제입니다.

이번 과제는 main 브랜치에서 직접 작업하지 않고, 작업 단위별로 feature 브랜치를 생성하여 진행합니다.

먼저 `feature/readme` 브랜치에서 전체 작업 계획과 브랜치 운영 방식을 README.md에 정리한 뒤, Pull Request를 생성하여 셀프 리뷰 후 main 브랜치로 Merge합니다.

이후 `feature/about` 브랜치에서는 자기소개 문서를 작성하고, `feature/skills` 브랜치에서는 현재 배우고 있는 기술과 관심 기술을 정리합니다.

각 작업은 브랜치 생성, 파일 수정, commit, push, Pull Request 생성, 셀프 리뷰, Merge 순서로 진행할 예정입니다.
